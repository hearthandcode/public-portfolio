# 04. ML boundaries and creative-model experiments

## What this directory shows

This directory records the difference between current work and future learning goals. It also includes a small, public-safe creative-model artifact to show how generative tools are used as part of a broader making practice.

## Artifacts

- `artifacts/suno-style-prompt-excerpt.md`: a style-only excerpt from a Suno music-generation prompt.
- `ml-and-retrieval-boundary.md`: current capability statement, learning direction, and claim boundary.

## Role in the wider architecture

The portfolio's current technical proof is bounded retrieval with provenance, not trained-model research. This directory makes that boundary visible rather than using the language of fine-tuning, reinforcement learning, or semantic embedding before there is an inspectable artifact.

## Boundary

The full lyric prompt is intentionally not copied because it includes personal material and is not required to demonstrate the creative workflow. No training run, fine-tuning result, RL system, semantic embedding result, or production ML evaluation is represented here.
